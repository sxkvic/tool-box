# 体重管理 · 饮食日记扩展设计

> 对标：薄荷健康 / Keep / 华为运动健康 的「饮食记录 + 宏量营养素」  
> 约束：纯前端、无后端；**不爬取**商业 App 接口（违法且不稳定）  
> 数据源：内置《中国食物成分表》风格本地库（每 100g）+ 用户可按克重换算

## 1. 市面主流能力（提炼）

| 产品 | 核心 |
|------|------|
| 薄荷健康 | 超大食物库、搜索/扫码、餐次、热量+碳水/蛋白/脂肪环形图、剩余可吃 |
| Keep / 乐刻类 | 饮食+运动消耗、目标热量 |
| 华为运动健康 | 体重趋势为主，饮食多为联动/简单记录 |
| MyFitnessPal | 品牌包装食品库、条码、宏量目标 |

共同路径：**选食物 → 填重量(g) → 按每100g营养素比例换算 → 按日/餐汇总 → 对比 TDEE/目标**

## 2. 本项目方案

### 能做（v1.1）
- 本地食物库 150+ 常见中餐/食材（主食、肉蛋奶、蔬果、零食饮品等）
- 搜索 + 分类浏览 + 最近/常用
- 按克重自动算：热量 kcal、蛋白质 g、脂肪 g、碳水 g
- 四餐：早/午/晚/加餐
- 今日仪表盘：已摄入 / 目标 / 剩余；三大营养素进度条
- 与体重档案联动：用身高体重年龄性别+活动量估 BMR/TDEE
- 目标模式：减脂 / 维持 / 增肌（自动推热量与宏量）

### 不做
- 爬取薄荷/食物库 API
- 条形码云识别、品牌包装全库
- 拍照 AI 识食物（可后续）

## 3. 换算公式

```
营养素 = per100 * (grams / 100)
kcal 可用 4*P + 9*F + 4*C 校验（以表内能量为准）
```

BMR（Mifflin-St Jeor）：
- 男：10W + 6.25H - 5A + 5
- 女：10W + 6.25H - 5A - 161

TDEE = BMR × 活动系数（1.2–1.9）

默认目标：
- 减脂：TDEE − 400
- 维持：TDEE
- 增肌：TDEE + 300

宏量默认（可调）：
- 减脂：P 30% / F 30% / C 40%
- 维持：P 20% / F 30% / C 50%
- 增肌：P 25% / F 25% / C 50%

## 5. 存储

`diet_hub_v1`：
```js
{
  entries: [{ id, date, meal, foodId, name, grams, kcal, protein, fat, carb, ts }],
  recentFoodIds: [],
  customFoods: [], // 用户自定义 per100
  prefs: { mode: 'lose'|'keep'|'gain', activity: '1.55', customKcal: '' }
}
```

体重档案仍在 `weight_hub_v1`。

## 6. 验收

2. 今日汇总三大营养素与剩余热量正确
3. 无网可用；主题与体重模块一致
