# 体重管理模块设计文档

> 对标：华为运动健康 · 体重管理  
> 形态：纯前端微信小程序 · 本地存储 · 双主题（深空 / 纯白）  
> 版本：v1.0

---

## 1. 产品定位

把现有「一次性 BMI 计算器」升级为可长期使用的 **体重管理中心**：
记录 → 目标 → 趋势 → 评估 → 建议。

核心原则：
- 本地隐私：体重数据只存本机 `wx.storage`，不上传
- 每日可记：记录体重不扣配额（FREE）
- 界面克制：参考华为运动健康的卡片信息密度，但沿用本项目 SpaceX HUD / Clean Lab 双主题
- 可落地：无后端、无设备同步、无付费会员

## 2. 功能范围（v1）

### 2.1 身体档案
- 性别、年龄、身高
- 用于 BMI / 理想体重 / 体脂估算

### 2.2 体重记录
- 快速记录：今日体重（kg，支持 0.1）
- 可选备注（晨起空腹、运动后等）
- 可选体脂率（手动输入，%）
- 同日多条：保留最新为「当日体重」，历史可进列表查看
- 编辑 / 删除

### 2.3 目标管理
- 目标体重
- 目标日期（可选）
- 自动计算：需减/增重量、剩余天数、建议周均变化
- 进度条（相对起始体重 → 目标体重）

### 2.4 趋势与统计
- 折线图：7 天 / 30 天 / 90 天 / 全部
- KPI：
  - 当前体重
  - 较上次变化
  - 近 7 天变化
  - 近 30 天变化
  - 总变化（相对首记）
- 最高 / 最低体重（当前区间）

### 2.5 健康评估
- BMI（中国成人标准）
- 状态：偏瘦 / 正常 / 偏胖 / 肥胖
- 理想体重区间（BMI 18.5–24）
- 简易体脂估算（若未手填）：Deurenberg 公式（仅供参考）
- 文案建议（非医疗建议免责）

### 2.6 与现有工具关系
- 保留独立 `BMI 计算` 工具（轻量单次计算）
- 新增 `体重管理` 作为重点入口（生活决策 / 健康）
- 体重管理内嵌 BMI 评估，避免割裂

## 4. 数据模型

Storage key：`weight_hub_v1`

```js
{
  profile: {
    gender: 'male' | 'female' | '',
    age: number | '',
    heightCm: number | ''
  },
  goal: {
    targetKg: number | '',
    deadline: 'YYYY-MM-DD' | '',
    startKg: number | '',      // 设定目标时的体重快照
    startDate: 'YYYY-MM-DD' | ''
  },
  records: [
    {
      id: string,              // 时间戳+随机
      date: 'YYYY-MM-DD',
      time: 'HH:mm',
      ts: number,
      weight: number,          // kg
      bodyFat: number | null,  // %
      note: string
    }
  ],
  prefs: {
    unit: 'kg',
    chartRange: '30'           // 7 | 30 | 90 | all
  }
}
```

规则：
- records 按 `ts` 降序
- 最多保留 800 条（够 2 年+每日）
- 写入前校验：30–300 kg，步长 0.1

## 5. 算法

### BMI
`BMI = weight / (heightm)^2`

中国成人：
- <18.5 偏瘦
- 18.5–23.9 正常
- 24–27.9 偏胖
- ≥28 肥胖

### 理想体重区间
`min = 18.5 * h^2`，`max = 24 * h^2`（kg，保留 1 位）

### 体脂估算（无手填时）
Deurenberg：  
`BF% = 1.2*BMI + 0.23*age - 10.8*sex - 5.4`  
sex：男=1，女=0。仅展示「估算」标签。

### 目标进度
- 若目标 < 起始：减重进度 = (start - current) / (start - target)
- 若目标 > 起始：增重进度对称
- clamp 0–100%，完成时 100%

### 建议周变化
`weekly = (current - target) / max(weeksLeft, 1)`  
安全提示：周变化建议控制在 0.25–1.0 kg（文案提示，不强制）

## 6. UI 设计

### 6.1 主页 Dashboard
1. **Hero 卡**：当前体重大数字 + 较上次 Δ + BMI 胶囊
2. **快捷记录按钮**：大主按钮「记录体重」
3. **目标卡**：目标体重、剩余、进度条、预计达成
4. **趋势卡**：Canvas 折线 + 渐变填充 + 区间 chip
5. **统计四宫格**：7 日 / 30 日 / 区间高低 / 总变化
6. **健康评估卡**：BMI 条、理想区间、体脂、建议
7. **最近 5 条记录** + 查看全部
8. 底部免责声明 + 广告位

### 6.2 视觉
- 深空：青光折线、HUD 卡片、数字等宽
- 纯白：白卡、蓝强调、轻阴影、无网格
- 体重数字 72–88rpx 粗体
- 涨红跌绿（减重用户常见心理：下降用 success 绿）

### 6.3 交互
- 首次进入无档案：引导去设置身高
- 无记录：空状态插画文案 + 一键记今日
- 记录弹层：体重输入 + 体脂可选 + 备注 chips + 保存
- 图表点选：显示当日体重气泡（简化：顶部摘要即可）
